# カテゴリ管理処理フロー・業務ロジック詳細

## 概要
カテゴリの追加・編集・削除の処理フローと業務ロジックを記載します。

---

## 1. 追加
- 管理者が `POST /api/categories` を呼び出し
- カテゴリ名を入力
- DB登録

## 2. 編集
- 管理者が `PATCH /api/categories/:id` を呼び出し
- 名称を更新

## 3. 削除
- 管理者が `DELETE /api/categories/:id` を呼び出し
- 紐づく写真の `category_id` は NULL になる
- DB論理削除（status=0）

## 業務ルール
- カテゴリ名は家族内で一意
- 削除時、紐づく写真の `category_id` は NULL 許容

## 未実装機能
- 並び順変更（カテゴリ数が少ないため優先度低）
